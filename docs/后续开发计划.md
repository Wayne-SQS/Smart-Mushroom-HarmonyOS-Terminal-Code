## 二、第二阶段：IoT 数据对接（第 1-2 周）

### 2.1 ViewModel 层搭建

| 序号 | 任务 | 产出文件 |
|------|------|----------|
| 2.1.1 | 创建 DashboardViewModel | `features/core/viewmodel/DashboardViewModel.ets` |
| 2.1.2 | 创建 DeviceViewModel | `features/core/viewmodel/DeviceViewModel.ets` |
| 2.1.3 | 创建 GrowthViewModel | `features/core/viewmodel/GrowthViewModel.ets` |
| 2.1.4 | 创建 ViewModel 基类 | `features/core/viewmodel/BaseViewModel.ets` |

**设计要点：**
- ViewModel 持有 `@State` 数据，Page 通过 `@Prop`/`@Link` 绑定
- 调用 `IotService` 获取真实数据
- 数据转换：IoT 原始影子数据 → UI 数据模型

### 2.2 华为云 IoTDA 对接

| 序号 | 任务 | 说明 |
|------|------|------|
| 2.2.1 | 配置设备凭证 | 在 `IotService.init()` 中填入 deviceId/secret/serverAddress |
| 2.2.2 | 定时轮询设备影子 | DashboardViewModel 每 10s 调用 `queryDeviceShadow()` |
| 2.2.3 | 属性值映射 | 将影子 `reported.properties` 映射到 `SensorDataItem` |
| 2.2.4 | 命令下发对接 | 快捷控制按钮调用 `sendCommand()` |
| 2.2.5 | 设备列表拉取 | DevicePage 对接真实设备列表 |
| 2.2.6 | 离线检测 | 调用 `isDeviceOnline()` 更新设备状态 |
| 2.2.7 | 错误处理 | 网络异常/鉴权失败时的 UI 降级展示 |

**数据流：**
```
IoTDA ← IotService → ViewModel → @State → Page UI
```

### 2.3 告警系统

**配方数据结构：**
```typescript

  mushroomType: string;      // 蘑菇品种
  growthStage: string;       // 生长阶段
  tempMin: number;           // 温度范围
  tempMax: number;
  humidityMin: number;       // 湿度范围
  humidityMax: number;
  co2Max: number;            // CO2上限
  lightMin: number;          // 光照范围
  lightMax: number;
}
```
