# SoftwareAction UI 美化实施方案（v2 详细版）

> 蘑菇大棚 IoT 管理系统界面升级 — 双主题 + 毛玻璃 + 动效
> 基于 HarmonyOS 官方设计规范 + 现代设计语言
> 更新日期：2026-06-28

---

## 一、设计目标

1. **双主题切换**：雾透玻璃（年轻活力）+ 极致黑白（专业克制）
2. **毛玻璃质感**：Header / 卡片 / TabBar 三处统一使用 `backdropBlur + 半透 + 白边`
3. **完整动效系统**：页面进入 / 卡片出现 / 按钮按下 / 数值过渡 / 状态脉冲
4. **零硬编码颜色**：所有颜色从 `ThemeManager.getTokens()` 读取，原子替换
5. **性能可控**：低端设备自动降级毛玻璃，限制同屏 backdropBlur 元素数量

## 二、双主题系统

> **颜色格式约定**：所有半透明色统一使用 8 位 hex（`#RRGGBBAA`），不使用 `rgba()`。
> ArkUI 对 `rgba()` 的解析在部分属性和 SDK 版本下不稳定，8 位 hex 是官方推荐格式。

### 2.1 主题 A — 雾透玻璃 🌸 `theme_frosted`

**关键词**：浅色、发白淡彩、毛玻璃、年轻、弹性动画

**视觉效果核心**：毛玻璃要"看得见"，依赖两个条件——
1. **下层有内容可模糊**：页面底色用渐变（`#E5EFE8` → `#F0F5F2`），Scroll 区域叠加极淡装饰纹理，卡片在不同滚动位置透出不同色调
2. **卡片透明度拉够**：半透白 `#FFFFFFA6`（≈65% 不透明），让下层渐变色透过来

#### 毛玻璃核心公式

> **blur vs backdropBlur（重要）**
> - `.backdropBlur(n)` — 模糊**下层内容**，毛玻璃效果用这个
> - `.blur(n)` — 模糊**组件自身内容**，装饰光晕用这个
> - 毛玻璃场景严禁用 `.blur()`

```typescript
// 雾透卡片 — 三件套：半透白底 + backdropBlur 模糊下层 + 白边
// 所有半透明色使用 8 位 hex（#RRGGBBAA），忌用 rgba()
Column() { /* 内容 */ }
  .borderRadius(16)
  .backgroundColor('#FFFFFFA6')   // 65% 白，透出下层渐变
  .backdropBlur(16)
  .border({ width: 1, color: '#59B4C8DC' })
  .shadow({ radius: 12, color: '#407EC8A3', offsetY: 4 })
```

> **为什么 backdropBlur 效果看得见？**
> 页面背景不是纯色，而是一个微妙的纵向渐变（`#E5EFE8` → `#F0F5F2`）——淡绿到暖白。
> 卡片是半透白 + 16vp 模糊，渐变在不同位置的色调透过卡片呈现微妙差异，
> 这才是"毛玻璃"的视觉效果来源。纯色背景 + 半透白 = 看起来只是灰色卡片。
>
> Scroll 区域可选叠加一层极淡的装饰纹理（见 3.2 节），进一步增强下层信息密度。

#### 动效规范

| 场景 | 实现 | 参数 |
|------|------|------|
| 页面进入 | `animateTo` opacity 0→1 + translateY 20→0 | 400ms springMotion(0.4, 0.8) |
| 卡片出现 | 逐个 stagger 弹出 | 每张间隔 60ms，scale 0.9→1.05→1.0 |
| 按钮按下 | 弹性反馈 | springMotion scale 1→0.92→1.0，200ms |
| 数值变化 | 数字滚动过渡 | animateTo 800ms EaseOut |
| 页面切换 | 淡入 + 上滑 | 300ms EaseInOut |
| 传感器状态 | 光晕脉冲（正常）/ 呼吸闪烁（告警） | 2s 循环 opacity 0.3↔0.6 |

### 2.2 主题 B — 极致黑白 ⚫ `theme_contrast`

**关键词**：纯黑纯白、高对比、无修饰、专业克制

**⚠️ 视觉关键**：卡片不能和页面背景同色，否则无边框无阴影时卡片完全不可见。

## 三、字体颜色安全规则 ⚠️

> 这是最容易出 Bug 的部分：之前多次出现"浅底配浅字"导致显示不清。

### 3.0.1 硬性规则

1. **正文/标题**在任何背景下对比度 ≥ 4.5:1（WCAG AA）
2. **提示/占位文字**对比度 ≥ 3:1（WCAG AA 大字号标准）
3. **传感器配色（`COLOR_TEMP` / `COLOR_HUMIDITY` 等）仅供装饰**——
   用于背景色、色条、指示点、图标。**严禁作为文字颜色直接使用。**
   需要显示传感器数值时，用 `TEXT_TITLE` / `TEXT_BODY`。
4. **按钮上文字**必须与按钮底色形成 ≥ 4.5:1 对比度。原则：
   - 深色按钮（绿/蓝/红/黑）→ 白色文字
   - 浅色按钮（黄/琥珀）→ 深色文字（`#3D2E00` 或 `#000000`）
5. **`TEXT_HINT` 只用在大字号场景**（≥ 14fp），小字号提示改用 `TEXT_SUB`

### 3.0.2 两主题文字色对比度矩阵

| Token | 雾透 hex | 雾透对比度(卡片底) | 黑白 hex | 黑白对比度(白卡底) |
|-------|----------|-------------------|----------|-------------------|
| `textTitle` | `#1A2A35` | **10.2:1** ✅ | `#000000` | **21:1** ✅ |
| `textBody` | `#324550` | **7.1:1** ✅ | `#1A1A1A` | **15:1** ✅ |
| `textSub` | `#546E7A` | **4.8:1** ✅ | `#555555` | **6.5:1** ✅ |
| `textHint` | `#7E929F` | **3.5:1** ⚠️≥14fp | `#777777` | **5:1** ✅ |
| `textWhite`（深底）| `#FFFFFF` | **12:1** on `#1B5E20` | `#FFFFFF` | **21:1** on `#000` |

> 雾透主题卡片底色 ≈ `#F1F5F3`（半透白叠加渐变后的有效色值），对比度按此计算。

### 3.0.3 传感器配色禁止用作文字色

```
❌ Text('22.5°C').fontColor('#F2B8B0')    // 淡珊瑚 on 白底 = 看不清
✅ Text('22.5°C').fontColor('#1A2A35')    // 用 textTitle
✅ 色条/背景 .backgroundColor('#F2B8B0') // 装饰用途 OK
```

### 4.2 传感器卡片 — 毛玻璃 + 状态光晕

```typescript
@Builder
SensorCard(item: SensorDataItem) {
  Stack() {
    // 状态光晕（装饰光晕用 .blur() 模糊自身）
    Ellipse()
      .width('80%')
      .height(30)
      .fill(this.getStatusGlowColor(item.status))
      .blur(20)
      .opacity(0.4)
      .position({ x: '10%', y: '70%' })

    Column() {
      // 图标 + 状态点
      Row() {
        Text(SensorConfigUtil.getConfig(item.type).icon).fontSize(22)
        Blank()
        Circle({ width: 8, height: 8 })
          .fill(this.getStatusColor(item.status))
          .shadow({ radius: 4, color: this.getStatusColor(item.status), offsetY: 0 })
      }
      .width('100%')

      // 数值 + 单位（大字号走 AdaptiveUtils 断点自适应，不走 token）
      Row() {
        Text(item.value.toFixed(1))
          .fontSize(AdaptiveUtils.getSensorDataFontSize(this.currentBreakpoint))
          .fontColor(this.t.textTitle)
          .fontWeight(FontWeight.Bold)
        Text(SensorConfigUtil.getConfig(item.type).unit)
          .fontSize(this.t.fontSm)
          .fontColor(this.t.textSub)
          .margin({ left: 4, bottom: 6 })
      }
      .alignItems(VerticalAlign.Bottom)
```

### 5.4 ThemeManager 单例

```typescript
// features/core/src/main/ets/theme/ThemeManager.ets
import { ThemeTokens } from './ThemeTokens';
import { FROSTED_THEME } from './ThemeFrosted';
import { CONTRAST_THEME } from './ThemeContrast';
import Preferences from '@ohos.data.preferences';
import { common } from '@kit.AbilityKit';

const PREFS_KEY = 'current_theme';
const PREFS_NAME = 'app_settings';

/**
 * 主题管理器单例
 *
 * 职责：
 * 1. 持有当前主题令牌
 * 2. 从本地存储恢复主题
 * 3. 切换主题并通知全局
 *
 * 用法：
 *   const t = ThemeManager.getInstance().getTokens();
 *   Text('hello').fontColor(t.textTitle)
 */
export class ThemeManager {
  private static instance: ThemeManager;
  private current: ThemeTokens = FROSTED_THEME;
  private prefs: Preferences | null = null;

  private constructor() {
    // 同步写入默认主题名，保证首帧不空
    AppStorage.setOrCreate<string>('themeName', 'frosted');
  }

  static getInstance(): ThemeManager {
    if (!ThemeManager.instance) {
      ThemeManager.instance = new ThemeManager();
    }
    return ThemeManager.instance;
  }

  /**
   * 异步从本地存储恢复上次主题
   *
   * 必须在 EntryAbility.onCreate 中 await 调用！
   * 不 await 会导致首帧显示默认 frosted，init 完成后二次重绘（闪变）。
   *
   * EntryAbility 调用方式：
   *   await ThemeManager.getInstance().init(this.context);
   */
  async init(context: common.Context): Promise<void> {
    this.prefs = await Preferences.getPreferences(context, PREFS_NAME);
    const saved = await this.prefs.get(PREFS_KEY, 'frosted') as string;
    const next = (saved === 'contrast') ? CONTRAST_THEME : FROSTED_THEME;
    if (next.name !== this.current.name) {
      this.current = next;
      AppStorage.setOrCreate<string>('themeName', this.current.name);
    }
  }

  /** 获取当前完整令牌 */
  getTokens(): ThemeTokens {
    return this.current;
  }

  /** 当前是否雾透主题 */
  isFrosted(): boolean {
    return this.current.name === 'frosted';
  }

  /** 当前是否黑白主题 */
  isContrast(): boolean {
    return this.current.name === 'contrast';
  }

  /**
   * 切换主题 — 原子替换
   *
   * 切换后写入本地存储 + 通知 AppStorage，所有 @StorageLink('themeName')
   * 的组件会自动触发 build() 重绘。
   */
  async toggle(): Promise<void> {
    const next = this.isFrosted() ? CONTRAST_THEME : FROSTED_THEME;
    this.current = next;
    if (this.prefs) {
      await this.prefs.put(PREFS_KEY, next.name);
      await this.prefs.flush();
    }
    AppStorage.setOrCreate<string>('themeName', next.name);
  }

  /**
   * 直接设置主题
   *
   * @param themeName 'frosted' 或 'contrast'
   */
  async setTheme(themeName: string): Promise<void> {
    const next = (themeName === 'contrast') ? CONTRAST_THEME : FROSTED_THEME;
    this.current = next;
    if (this.prefs) {
      await this.prefs.put(PREFS_KEY, next.name);
      await this.prefs.flush();
    }
    AppStorage.setOrCreate<string>('themeName', next.name);
  }
}
```

### 5.7 Motion Token（动效令牌）

**动机**：当前动画参数散落在第八章各示例中（`duration: 300`、`duration: 800`、`duration: 2000`、`Curve.EaseOut` 等），改节奏要改 N 处。统一抽成 MotionToken 后，整个 App 的动画节奏完全一致，调一个值全局生效。

#### 5.7.1 MotionTokens 接口

```typescript
// features/core/src/main/ets/theme/MotionTokens.ets
import { Curve } from '@kit.ArkUI';

/**
 * 动效令牌 — 统一管理全 App 动画时长与曲线
 *
 * 命名规范：用语义而非数值（fast/normal/slow），方便主题切换时整体调速。
 * 黑白主题可关闭动画（durations 全为 0），但接口不变。
 */
export interface MotionTokens {
  // 时长（毫秒）
  readonly fast: number;            // 150ms — 按钮按下、toggle、颜色瞬切
  readonly normal: number;          // 300ms — 标准过渡、页面切换
  readonly slow: number;            // 400ms — 页面进入、卡片出现单张
  readonly dataTransition: number;  // 800ms — 数值滚动过渡
  readonly stagger: number;         // 60ms — 卡片出现间隔
  readonly breath: number;          // 2000ms — 告警呼吸循环

  // 曲线
  readonly spring: Curve;           // 弹性曲线 — 按钮按下、卡片弹出
  readonly easeOut: Curve;          // 减速曲线 — 数值过渡、进入
  readonly easeInOut: Curve;        // 平滑曲线 — 页面切换、呼吸

  // 动画开关（黑白主题=false）
  readonly enabled: boolean;
}
```

#### 5.7.2 两个主题的 MotionToken 实例

```typescript
// 雾透玻璃 — 弹性、有节奏
export const FROSTED_MOTION: MotionTokens = {
  fast: 150,
  normal: 300,
  slow: 400,
  dataTransition: 800,
  stagger: 60,
  breath: 2000,
  spring: Curve.Spring,
  easeOut: Curve.EaseOut,
  easeInOut: Curve.EaseInOut,
  enabled: true
};

// 极致黑白 — 关闭所有动画，保留最低反馈
export const CONTRAST_MOTION: MotionTokens = {
  fast: 50,           // 仅保留 50ms 颜色瞬切
  normal: 0,
  slow: 0,
  dataTransition: 0,
  stagger: 0,
  breath: 0,
  spring: Curve.Linear,
  easeOut: Curve.Linear,
  easeInOut: Curve.Linear,
  enabled: false
};
```

#### 5.7.3 挂载到 ThemeManager

```typescript
// ThemeManager 增加获取 motion 的方法
export class ThemeManager {
  private currentMotion: MotionTokens = FROSTED_MOTION;

  getMotion(): MotionTokens {
    return this.currentMotion;
  }

  async toggle(): Promise<void> {
    // ... 原有逻辑 ...
    this.currentMotion = this.isFrosted() ? CONTRAST_MOTION : FROSTED_MOTION;
  }

  async setTheme(themeName: string): Promise<void> {
    // ... 原有逻辑 ...
    this.currentMotion = (themeName === 'contrast') ? CONTRAST_MOTION : FROSTED_MOTION;
  }
}
```

#### 5.7.4 页面使用方式

```typescript
@Component
export struct DashboardPage {
  private get t(): ThemeTokens { return ThemeManager.getInstance().getTokens(); }
  private get m(): MotionTokens { return ThemeManager.getInstance().getMotion(); }

  // 控制按钮脉冲动画
  .onClick(() => {
    if (this.m.enabled) {
      animateTo({
        duration: this.m.normal,        // 300ms
        curve: this.m.spring,            // Curve.Spring
        onFinish: () => { this.pulseScale = 1; }
      }, () => { this.pulseScale = 1.3; });
    } else {
      this.pulseScale = 1.3;             // 黑白主题直接跳变
    }
  })
}
```

> **原则**：页面中严禁出现裸数字 `duration: 300` 或裸曲线 `Curve.EaseOut`，必须从 `this.m` 读取。新增动画场景时，先在 MotionTokens 定义语义字段，再在两个主题中实例化。

> **统一来源**：本章所有 `duration` 与 `curve` 必须从 `this.m`（MotionTokens）读取，严禁裸数字。语义对照：`fast=150ms` / `normal=300ms` / `slow=400ms` / `dataTransition=800ms` / `stagger=60ms` / `breath=2000ms`。两主题各自实例化（雾透用弹性曲线，黑白用 Linear/0ms），调一个值全局生效。

### 8.4 卡片出现动画（stagger）

```typescript
@State cardOpacity: number[] = [0, 0, 0, 0, 0, 0];
private staggerTimers: number[] = [];  // 用于清理

aboutToAppear() {
  for (let i = 0; i < 6; i++) {
    const timer = setTimeout(() => {
      animateTo({ duration: this.m.slow, curve: this.m.easeOut }, () => {  // 400ms
        // ⚠️ ArkTS V1 不追踪数组元素赋值，必须替换整个数组
        const next = [...this.cardOpacity];
        next[i] = 1;
        this.cardOpacity = next;
      });
    }, i * this.m.stagger);  // 60ms
    this.staggerTimers.push(timer);
  }
}

aboutToDisappear(): void {
  // ⚠️ 必须清理：组件销毁后 setTimeout 仍会触发，导致框架报错
  for (let i = 0; i < this.staggerTimers.length; i++) {
    clearTimeout(this.staggerTimers[i]);
  }
  this.staggerTimers = [];
}
```

### 8.5 状态光晕呼吸（告警时）

```typescript
@State glowOpacity: number = 0.3;
private animatingGlow: boolean = false;

startGlowAnimation(): void {
  if (this.animatingGlow) { return; }  // 防止重复启动
  this.animatingGlow = true;
  this.glowLoop();
}

private glowLoop(): void {
  if (!this.animatingGlow) { return; }  // 停止条件
  animateTo({
    duration: this.m.breath,       // 2000ms
    curve: this.m.easeInOut,        // Curve.EaseInOut
    iterations: 1,            // ⚠️ 不要用 iterations: -1，组件销毁后无法停止
    playMode: PlayMode.Normal,
    onFinish: () => {
      // 交换目标值，手动循环
      this.glowOpacity = this.glowOpacity > 0.4 ? 0.3 : 0.6;
      this.glowLoop();
    }
  }, () => {
    this.glowOpacity = this.glowOpacity > 0.4 ? 0.3 : 0.6;
  });
}

stopGlowAnimation(): void {
  this.animatingGlow = false;
}

aboutToDisappear(): void {
  this.stopGlowAnimation();  // ⚠️ 必须停止，否则继续持有 this 引用 → 内存泄漏
}
```

## 九、迁移策略 —— 从 StyleConstants 到 ThemeManager

> 核心问题：现有 ~600 行 DashboardPage 全部引用 `StyleConstants.XXX`，不能一次性全部改完。

### 9.1 桥接模式：让 StyleConstants 变成 ThemeManager 的代理

不改任何现有页面代码，先让 `StyleConstants` 转发到 `ThemeManager`：

```typescript
// common/base/src/main/ets/constants/StyleConstants.ets
// 原有静态字段改为 getter，从 ThemeManager 读取当前主题 token
import { ThemeManager } from 'features-core';  // ❌ 违反依赖方向！

// ⚠️ common-base 不能 import features-core。改为在 features-core 中
// 提供 init 函数，AppStorage 写入后覆盖。
```

由于三层架构的单向依赖约束（common-base 不能依赖 features-core），桥接方案改为：

**在 `AppStorage` 中同时存储 theme tokens 的序列化副本**，`common-base` 的组件通过 `@StorageLink` 读取。

### 9.2 推荐方案：逐页迁移 + 过渡期内两套共存

1. **第一批**：创建 ThemeManager + 主题文件，不改 StyleConstants
2. **第二批**：选一个页面（建议 DashboardPage）作为试点，全量改为 `this.t`
3. **验证**：试点页面主题切换正常，无闪白，颜色正确
4. **逐页推进**：每改完一个页面就测试，保证不出现半页新旧混色
5. **最后**：全部页面改完后，`StyleConstants` 保留为常量参考（不再被引用）

### 9.3 hex 颜色 + alpha 后缀问题

现有代码大量使用此模式：
```typescript
.backgroundColor(item.color + '20')      // '#7EC8A3' + '20' = '#7EC8A320' ✅ OK
.backgroundColor(StyleConstants.DANGER + '18')  // '#FF4D4F' + '18' = '#FF4D4F18' ✅ OK
```

迁移到 ThemeManager 后，所有颜色已是完整 8 位 hex，**不要再拼 alpha 后缀**：
```typescript
// ❌ 迁移后错误
.backgroundColor(this.t.primary + '20')  // '#7EC8A3' + '20' = '#7EC8A320' → 仍 OK（巧合）

// ✅ 迁移后正确做法：预定义带透明度的 token
.backgroundColor(this.t.bgCardLight)     // '#FFFFFF66' → 40% 白
```

需要在 `ThemeTokens` 接口和两个主题常量中预定义所有需要的透明度变体，
避免运行时字符串拼接。如果确有需要动态透明度，使用 `Color.argb()` API。

### 9.4 AdaptiveUtils 新增方法

在迁移前，需要先在 `BreakpointConstants.ets` 中新增 `AdaptiveUtils` 方法：

```typescript
// 概览卡片数字字号（比传感器数据更大一级）
static getOverviewFontSize(breakpoint: string): number {
  switch (breakpoint) {
    case BreakpointConstants.XS: return 20;
    case BreakpointConstants.SM: return 24;
    case BreakpointConstants.MD: return 28;
    case BreakpointConstants.LG: return 32;
    case BreakpointConstants.XL: return 36;
    default: return 24;
  }
}
```

## 十二、关键约束（不可违反）

1. **零硬编码颜色**：组件中不允许出现 `'#XXXXXX'`，必须从 `this.t` 读取
2. **禁止 hex 颜色字符串拼接 alpha**：迁移到 ThemeManager 后，所有颜色已是完整 8 位 hex，
   `this.t.primary + '20'` 不再被允许。需要不同透明度 → 在 Token 中预定义或使用 `Color.argb()`
3. **Token 完整性**：每新增一个颜色/数值，必须同时在两个主题中定义
4. **原子替换**：主题切换时整组 Token 替换，不允许部分替换
5. **性能优先**：低端设备 backdropBlur 自动降级，动画可关闭
6. **无 any 类型**：ThemeTokens 接口字段必须明确类型
7. **动画只动复合属性**：`scale / translate / opacity`，禁止动画 `width / height / margin`
8. **毛玻璃用 `backdropBlur`，光晕用 `blur`**：
   - 卡片/Header/TabBar 的毛玻璃效果 → `.backdropBlur(this.t.blurAmount)`（模糊下层内容）
   - 装饰光晕（Ellipse 模糊自身边缘）→ `.blur(n)`（模糊自身）
   - 严禁混用，严禁毛玻璃场景用 `blur`
9. **字号分两级管理**：
   - **小字号场景**（标题/正文/辅助文字/单位/标签）→ 用 `this.t.fontXs/fontSm/fontMd/fontLg/fontXl`
   - **大字号场景**（传感器数值/概览数字/图表数据）→ 用 `AdaptiveUtils.getXxxFontSize(breakpoint)` 断点自适应
   - 严禁大字号场景用 token 固定字号（手机端会过大）
10. **字体对比度安全**（⚠️ 最容易踩坑）：
   - 正文/标题 vs 底色对比度 ≥ 4.5:1，提示文字 ≥ 3:1
   - 传感器配色（`COLOR_TEMP` 等淡色）**仅供装饰**，严禁作为文字颜色
   - 按钮上文字：深色按钮用白色，浅色按钮用深色
   - 每改一个颜色，先在两主题的所有底色上验证可读性
11. **backdropBlur 条件化**：`blurAmount === 0` 时不设置 `.backdropBlur()`，避免无效渲染开销
12. **动画生命周期**：`animateTo({ iterations: -1 })` 禁止使用，改为手动循环；`setTimeout` 必须在 `aboutToDisappear` 中 `clearTimeout`
13. **`@State` 数组更新**：ArkTS V1 不追踪数组元素赋值 `arr[i] = val`，必须替换整个数组 `this.arr = [...this.arr.slice(0, i), val, ...this.arr.slice(i + 1)]`
14. **颜色格式**：所有半透明色使用 8 位 hex（`#RRGGBBAA`），不使用 `rgba()`
